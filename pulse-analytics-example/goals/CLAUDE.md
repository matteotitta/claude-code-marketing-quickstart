# marketing/goals/ — KPIs + measurement + sprint cycles

Holds the **goals + KPIs** for the marketing OS — what you're measuring, what good looks like, how you'll know the system is working.

## Empty in V1 (you populate as you go)

The structure when populated:

| File | Purpose |
|---|---|
| `MMYY-scope.md` | KPIs scoped from your current planning cycle (definitions, baseline, targets, deadlines) |
| `measurement.md` | How each KPI gets measured (sources: GA4, GSC, CRM, etc.; cadence; owner) |
| `cadence.md` | Sprint length (1 or 2 weeks) + notification preferences |
| `MMYY-NN-cycle.md` | Per-sprint cycle file (results + hypotheses; NN = sprint number) |

## Why this folder exists

Marketing OS without measurable goals decays into "we shipped a lot of content." Goals tie skill output back to outcomes the business cares about — pipeline, retention, CAC payback, brand mentions.

**Read goals before producing any execution output.** If you're about to write a landing-page draft, check `goals/MMYY-scope.md` first — does this work hit a KPI on the current scope?

## Refresh cadence

- `MMYY-scope.md` quarterly (or per planning cycle)
- `MMYY-NN-cycle.md` per sprint (1 or 2 weeks)
- `measurement.md` + `cadence.md` quarterly review
