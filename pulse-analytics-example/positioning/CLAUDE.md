# marketing/positioning/ — positioning strategy (canonical)

Holds the **canonical positioning** for your product — anchor, differentiators, status-quo alternatives, value propositions. One locked file per refresh cycle.

## What's in here

| File | Status | Produced by |
|---|---|---|
| [`positioning.md`](./positioning.md) | EXAMPLE — PulseAnalytics positioning; replace in this repo | `/positioning` |

When you refresh quarterly, you supersede with a new dated file (e.g., `0826-positioning.md`) and archive the prior version. Genesys convention: `MMYY-topic.md` naming.

## What downstream skills read this

- `/product-messaging` — needs positioning statement + anchors as inputs
- `/landing-page-copy` — needs hero positioning + status-quo alternatives
- `/linkedin-content` — needs anchor + key differentiators for hook design
- `/ad-creative-brief` — needs status-quo alternatives + differentiators

## Refresh cadence

Quarterly. Trigger sooner if competitor moves invalidate the anchor or ICP feedback signals positioning is landing differently than expected.
